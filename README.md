# 二手房 FCE 评分器

一个本地优先的 React/Vite 工具，用于录入二手房房源、配置指标体系、统一评分，并通过 FCE（模糊综合评价）+ AHP（层次分析法）权重辅助筛选候选房源。

**所有数据存储在浏览器 localStorage，不上传服务器。**

公开页面：

```text
https://enhk2728-ui.github.io/second-hand-home-scorer/
```

## 首次使用

首次打开本工具时会显示引导向导，包含：

- **风险声明与免责**：本工具是二手房比较与决策辅助，不构成购房、金融、法律或税务建议。请自行核验产权、贷款、税费、学区/政策、房屋质量和交易风险，必要时咨询专业人士。
- **快速上手指南**：录入房源 → 选择/调整指标 → 设置权重/一致性矩阵 → 评分 → 查看排序 → 导出备份。
- **指标升级提示**（如适用）：检测到旧版指标配置时，可选择合并新版默认指标、保留现有配置或重置为示例数据。

你可以随时通过左侧边栏底部的「帮助」按钮重新打开引导。

## 能做什么

- 房源基础信息录入与编辑（名称、位置、总价、面积、单价）
- 统一批量打分（跨房源对比评分）
- 可编辑指标体系（21 个默认指标，可增删改）
- AHP 自动权重（等权 / 按重要性排序）
- AHP 手动判断矩阵（Saaty 1-9 标度）
- 几何平均法计算权重 + 一致性检验
- FCE 模糊综合评价（优秀/良好/一般/较差 四档隶属度）
- 硬伤否决（产权风险、结构硬伤等自动淘汰）
- 结果排序、雷达图、热力图
- JSON 备份/恢复（兼容旧版导入）
- CSV 导出（含 BOM UTF-8，Excel 直接打开）

## 不能做什么

- **不是房产估值工具**——不计算市场价或投资回报率
- **不是 AI 推荐系统**——分数基于你设定的权重和规则
- **不联网**——所有操作在浏览器内完成，无后端

## 旧用户注意事项

### 为什么我看到的还是旧指标？

旧版默认指标为 18 个，当前版本已扩充至 21 个。如果你的浏览器之前使用过旧版本，`localStorage` 中保存了旧的指标配置，`loadState()` 会优先读取本地存储的数据，因此你可能看不到新增的默认指标。

### 如何获取新版默认指标？

有以下几种方式：

1. **合并新版默认指标（推荐）**：打开应用后，如果检测到旧版指标，引导向导会提示你"合并新版默认指标"。此操作会将缺失的新版指标追加到现有指标列表末尾，**不会删除你的房源、评分或自定义设置**。

2. **手动重置**：在「指标」页面点击「重置默认」按钮，将指标列表恢复为最新默认值。

3. **手动添加**：在「指标」页面点击「新增指标」手动创建你需要的指标。

4. **清除浏览器数据后刷新**：清除该域名的 `localStorage` 后，应用将自动加载最新默认配置。**注意：此操作会丢失所有浏览器本地数据，请先导出 JSON 备份。**

## 推荐使用流程

### 1. 添加/编辑房源基本信息

进入「录入」页面，填写：

- **房源名称**：例如 "A 房源"、"梧桐里 3栋"
- **位置**：简要地址或区域
- **总价（万）**：例如 `520`（万）
- **面积（m²）**：例如 `89`
- **单价（元/m²）**：自动根据 `总价×10000 ÷ 面积` 计算

### 2. 选择并配置指标

进入「指标」页面，查看预设的 21 个指标。可以：

- 增删指标、修改名称和分类
- 勾选/取消"参与评分"（不参与评分的指标不影响最终得分）
- 勾选"硬伤否决"（此类指标为真时，该房源直接淘汰）

### 3. 选择权重模式

在指标页面的「AHP 权重配置」区域，选择三种模式之一：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **自动：等权** | 所有参与评分的指标权重平均分配 | 快速比较，没有特别偏好 |
| **自动：按重要性排序** | 拖排指标顺序，系统自动生成判断矩阵 | 有明确的优先级但不纠结具体倍数 |
| **手动：判断矩阵** | 手动填写两两比较值（Saaty 1-9） | 需要精细控制权重比例 |

### 4. 检查一致性

无论哪种模式，系统都会显示一致性指标：

- **λmax**（最大特征值）、**CI**（一致性指标）、**RI**（随机指数）、**CR**（一致性比率）
- **通过**：CR ≤ 0.1，权重可靠
- **不通过**：CR > 0.1，建议调整判断矩阵（自动模式通常不会出现此情况）

### 5. 应用权重并打分

点击「应用权重」后，进入「打分」页面，对所有房源逐项评分：

- **1-10 评分**：拖动滑块
- **数字输入**：直接输入数值（如通勤时间 35 分钟）
- **四档选择**：从下拉框选择（如 "理想楼层"）
- **是否项**：勾选（硬伤否决类勾选后直接淘汰）

### 6. 查看结果

进入「结果」页面可以看到：

- **综合排名**：按总分降序
- **各档隶属度**：优秀/良好/一般/较差 柱状图
- **雷达图**：多维度可视化
- **热力图**：房源×指标矩阵
- **淘汰列表**：含硬伤否决的房源

### 7. 导出数据

顶部工具栏：
- **导出 JSON**：备份所有数据（房源、指标、配置）
- **导入 JSON**：恢复备份（兼容旧版格式）
- **导出 CSV**：用 Excel 打开房源×指标得分表

## 默认指标体系

当前版本包含 **21 个**默认指标，按 5 个类别组织：

### 位置通勤（4 项）
通勤时间、地铁/公交便利、周边生活便利、学区/教育匹配

### 价格价值（4 项）
总价压力、单价合理性、议价空间、未来流动性

### 房屋本体（5 项）
户型满意度、采光通风、楼层/电梯、装修维护成本、噪音影响

### 小区物业（4 项）
小区品质、物业服务、停车便利、环境与安全

### 风险流动性（4 项）
产权/交易硬伤、房屋结构/漏水硬伤、抵押查封风险、税费/交易周期压力

## 指标类型详解

### 四种输入类型

| 类型 | 说明 | 评分方式 | 适合指标 |
|------|------|----------|----------|
| **1-10评分** | 1-10 主观评分 | 滑块选择 1-10 | 户型满意度、采光通风、物业服务 |
| **数字** | 具体数值 | 输入数字 | 通勤时间（分钟）、装修成本（万） |
| **四档选择** | 优秀/良好/一般/较差 | 下拉框选择 | 学区匹配、楼层/电梯、税费情况 |
| **是否项** | 布尔（是/否） | 勾选 | 产权硬伤、结构硬伤、查封风险 |

### FCE 规则说明

每种指标类型有对应的 FCE 规则：

- **1-10评分**：9-10→优秀、7-8→良好、5-6→一般、1-4→较差
- **数字**：可自定义阈值区间（如通勤≤25分钟→优秀、25-40→良好等）
- **四档选择**：每个选项映射到一个评价等级
- **是否项（硬伤）**：勾选"是"→硬伤否决；不参与评分。非硬伤的是否项："是"→优秀，"否"→不计分

### 硬伤否决 vs 评分指标

- **硬伤否决指标**（`isHardFailCapable: true`）：如产权风险、结构硬伤。勾选后该房源直接淘汰，总分归零。
- **评分指标**（`participatesInScoring: true`）：正常参与 FCE 评分。
- 一个指标可以同时是硬伤否决和参与评分（如需要），但默认硬伤否决指标不参与评分。

## AHP 判断矩阵教程

### Saaty 1-9 标度含义

在两两比较中，"A 比 B 重要多少" 使用以下标度：

| 标度 | 含义 | 解释 |
|------|------|------|
| 1 | 同等重要 | A 和 B 同样重要 |
| 3 | 稍微重要 | A 比 B 稍微重要一些 |
| 5 | 明显重要 | A 比 B 明显重要 |
| 7 | 强烈重要 | A 比 B 强烈重要 |
| 9 | 极端重要 | A 比 B 极端重要 |
| 2,4,6,8 | 中间值 | 介于上述等级之间 |
| 1/2, 1/3...1/9 | 倒数 | A 不如 B 重要（取对应正数的倒数） |

### 示例：通勤时间 vs 单价合理性

假设你认为「通勤时间」比「单价合理性」稍微重要（标度 3）：

- 在矩阵的「通勤时间」行和「单价合理性」列交叉处选择 `3`
- 系统自动在对角对称位置填入 `1/3`

### 为什么倒数值自动填充？

AHP 判断矩阵必须是正互反矩阵：如果 A 比 B 重要 3 倍，那么 B 比 A 重要 1/3 倍。系统自动填充倒数值，确保矩阵一致性。

### 上三角 vs 下三角

在手动模式中：
- **上三角**（行指标 vs 列指标）：可编辑的下拉框
- **对角线**：固定为 1（指标与自身比较）
- **下三角**：只读，显示对应上三角的倒数

## 一致性检验教程

### 什么是 AHP 一致性？

当你填写 AHP 矩阵时，你的判断可能隐含矛盾。例如：

> 通勤 > 单价（标度 3）
> 单价 > 户型（标度 5）
> → 按逻辑，通勤应该 >> 户型（约 15）
> 但如果你填了通勤 = 户型（标度 1），这就是不一致。

### CR（一致性比率）

- **λmax**：最大特征值，反映矩阵的整体偏离程度
- **CI** = (λmax − n) / (n − 1)：一致性指标，n 是指标数量
- **RI**：随机一致性指数（Saaty 的标准表格）
- **CR** = CI / RI：**一致性比率**

### 实用规则

- **CR ≤ 0.1**：通过检验，权重可信
- **CR > 0.1**：不通过，建议调整

### CR 不通过怎么办？

1. 找出最矛盾的比较（检查哪些值明显不合理）
2. 减少比较数量（减少指标更容易通过）
3. 改用「自动：按重要性排序」模式（系统保证一致性）
4. 如果认为判断没问题，可以强制应用（但结果可能不稳定）

**注意：** 等权模式和排序模式生成的矩阵天然具有较高一致性，通常 CR 非常低。

## 备份与导出指南

### JSON 备份（推荐）

- 点击「导出 JSON」下载 `.json` 文件
- 包含所有房源、指标、评分、AHP 配置
- 可在任何设备通过「导入 JSON」恢复
- **兼容旧版备份**：缺少新字段的旧备份会自动补充默认值

### CSV 导出

- 点击「导出 CSV」下载 `.csv` 文件
- 包含房源×指标矩阵 + 综合分 + 淘汰状态
- 用 Excel 打开即可，含 BOM 确保中文不乱码

### 数据安全

- 所有数据存储在浏览器 `localStorage`
- 清除浏览器数据会丢失当前状态
- **建议定期导出 JSON 备份**

- Node.js 18+
- npm（随 Node.js 安装）



