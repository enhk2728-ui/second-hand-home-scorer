# TASK-047 - 指标升级引导、首次教程声明与 UI 美化

## 背景

仓库：`C:\Users\Administrator\PycharmProjects\projects\second-hand-home-scorer`

用户反馈：打开项目网站后仍看到原有指标，怀疑上一轮没有扩充指标。当前代码里 `DEFAULT_INDICATORS` 已扩充，但 `src/domain/storage.ts` 的 `loadState()` 优先读取 `localStorage` 中的 `second-hand-home-scorer-state-v1`。因此老用户或同一浏览器之前保存过旧状态时，会继续看到旧指标。这是数据迁移/引导缺失，不应通过静默覆盖用户数据解决。

## 目标

在不改变现有评分核心逻辑的前提下，完成三件事：

1. 让旧浏览器状态能发现并合并新版默认指标。
2. 本地部署或网页打开后，首次进入先出现教程和风险声明引导，并可再次打开。
3. 美化整体 UI、动效和平面设计，保持工具型应用体验，不做营销落地页。

## 功能要求

### 1. 旧状态升级与默认指标合并

实现非破坏性的默认指标升级流程：

- 检测当前加载状态是否缺失 `DEFAULT_INDICATORS` 中的部分默认指标。
- 不要静默覆盖用户已有指标、房源、评分、权重。
- 给用户明确入口：
  - `保留现有指标`：不合并，只记录用户已知晓。
  - `合并新版默认指标`：追加缺失的默认指标，保留现有数据。
  - `重置为新版示例数据`：必须有明确确认，使用 `DEFAULT_STATE`，会覆盖当前浏览器数据。
- 合并时保留已有指标顺序，缺失默认指标追加到后面；已有房源的新增指标值可以为空/未评分。
- 合并后应让权重配置仍可用，不能造成总分 NaN、空白页或控制台错误。
- 建议新增独立模块，例如 `src/domain/stateMigration.ts`，并配套测试。
- 可以新增 localStorage 标记，例如 `second-hand-home-scorer-onboarding-v2` 或默认指标版本标记；命名保持清晰。

### 2. 首次教程与声明引导

新增首次打开引导 UI，满足：

- 首次打开或检测到旧指标状态时自动出现。
- 用户可以从页面顶部或侧边栏通过帮助/信息按钮再次打开。
- 内容至少包括：
  - 本工具是二手房比较与决策辅助，不构成购房、金融、法律、税务建议。
  - 请自行核验产权、贷款、税费、学区/政策、房屋质量和交易风险，必要时咨询专业人士。
  - 数据仅保存在当前浏览器 `localStorage`，不上传服务器。
  - 基本流程：录入房源 -> 选择/调整指标 -> 设置权重/一致性矩阵 -> 评分 -> 查看排序 -> 导出备份。
  - 说明旧用户若看不到新版指标，可以使用“合并新版默认指标”。
- 引导要是应用内工具的一部分，不要做成营销首页。
- 交互文案使用中文，专业、简洁。

### 3. UI 与动效美化

在不改变主要业务逻辑的前提下重构视觉层：

- 设计方向：专业的房源研究工作台/评分驾驶舱，信息密度适中，适合反复录入和比较。
- 不要做营销 hero，不要大段功能介绍。
- 避免单一色系，尤其避免页面被紫色、米色/沙色、深蓝/石板色、棕橙色完全主导。
- 改善整体层级：
  - 应用外框、侧边栏、顶部状态区。
  - 卡片、表格、表单、按钮、分段控件、空状态。
  - 指标设置、AHP 一致性矩阵、评分卡、结果排序和导出区域。
- 使用已有 `lucide-react` 图标补强工具按钮和帮助入口。
- 加入 CSS 动效：
  - 首屏/面板渐入。
  - 导航 active 状态过渡。
  - 按钮 hover/press。
  - modal 进入/退出。
  - 重要提示轻微动效。
- 必须支持 `prefers-reduced-motion`。
- 移动端和桌面端不能出现文字重叠、按钮文字溢出或布局跳动。
- 保持固定格式控件尺寸稳定，例如矩阵单元格、工具按钮、指标标签、评分结果列表。
